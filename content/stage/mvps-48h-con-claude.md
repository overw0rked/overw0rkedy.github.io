El gap entre "tengo una idea" y "tengo algo que puedo enseñar a un cliente" históricamente eran semanas. Con Claude como copiloto técnico, son **48 horas** — si sabes qué construir, qué evitar, y cómo pedirlo.

Workshop intensivo de fin de semana donde cada participante sale con un MVP corriendo en producción. No es teórico: traes una idea, te vas con landing + backend + base de datos + auth + deploy.

## 01 Lo que vamos a hacer

- **Día 1:** validar la idea con 10 conversaciones + diseñar la arquitectura mínima con Claude.
- **Día 2:** construir, deployar a Vercel, conseguir tu primer usuario beta antes de irte.
- **Stack base:** Next.js · Supabase · Stripe · Vercel — todo prompteado contra Claude.

## 02 Agenda

| Día / hora | Bloque |
|---|---|
| **SÁB 09:00** | Kickoff · idea pitch (60 seg c/u) |
| **SÁB 10:00** | Validation sprint · 10 conversaciones |
| **SÁB 14:00** | Arquitectura con Claude · schema, endpoints, data model |
| **SÁB 18:00** | Build sprint #1 · backend + DB + primer endpoint |
| **DOM 09:00** | Build sprint #2 · frontend, auth, Stripe sandbox |
| **DOM 15:00** | Deploy + primer usuario · Vercel push, compartir link |
| **DOM 18:00** | Demo & retrospectiva |

## 03 Quién debería venir

Founders con una idea concreta que quieran validar sin esperar a contratar dev. Necesitas: laptop, una idea que ya tengas masticada mentalmente, y disponibilidad completa el fin de semana del 10-11 de junio.

**Cupos limitados a 15 personas.** Costo y registro por confirmar.
